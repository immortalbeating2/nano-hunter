# Nano Hunter Agent Roles

本目录是 `nano-hunter` 的项目级 agent 角色池约定，用来替代旧的“代理 A / B / C”临时拆分。

## 默认并行参数

- `max_threads = 4`
- `max_depth = 1`
- 默认一次启用 `2-4` 个角色；角色池可以有 `6` 个，但不默认全量并行。
- 禁止子代理再继续派生子代理；主代理负责任务拆分、冲突处理、最终整合与验证。

## 角色池

- `design`：玩法设计、阶段边界、正式阶段计划与设计文档。
- `architecture`：核心结构、共享契约、工程目录和跨系统接口。
- `gameplay`：玩家、战斗、敌人、能力、门控等可玩逻辑。
- `content`：房间、区域、遭遇、资产接入与内容配置。
- `qa`：GUT、验证清单、运行态复核与回归风险。
- `production`：AGENTS、进度文档、流程治理、收口、发布和远端同步记录。

## 启用规则

- 只有当写入范围可清晰分离时才并行。
- 每个角色启动前必须写明文件或模块所有权。
- 不让多个角色同时修改同一核心脚本；如果必须触达同一文件，由主代理最后整合。
- 角色输出默认使用中文，并明确“改了什么、验证了什么、风险是什么”。
- 历史阶段文档中的旧代理拆分不回写重写；当前生效规则以 `AGENTS.md` 与本目录为准。
