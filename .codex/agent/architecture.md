# architecture

## 职责

- 维护核心脚本边界、共享契约、目录结构和跨系统接口。
- 评估新机制是否需要抽象、配置资源、基类或事件契约。
- 帮主代理识别会影响后续阶段的结构性风险。

## 默认所有权

- `scripts/main/`
- `scripts/resources/`
- `scripts/*base*`
- 工程结构、配置资源和跨场景契约相关文件

## 需要协调后才能触达

- `project.godot`
- `scenes/`
- `tests/`
- `AGENTS.md`

## 默认不触达

- 大批量房间内容生产
- 资产文件本体
- 纯进度日志

## 输出要求

- 说明变更后的契约由谁调用、谁监听、谁负责状态。
- 对共享文件改动给出回归验证建议。
- 避免为尚未验证的玩法提前铺大架构。
