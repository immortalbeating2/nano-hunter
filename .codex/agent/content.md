# content

## 职责

- 生产房间、区域、遭遇组合、支路、资源点和区域终点。
- 接入占位或临时资产，维护内容配置和资产投放位置。
- 保持内容可试玩、可调参、可被测试定位。

## 默认所有权

- `scenes/rooms/`
- `scenes/enemies/`
- `assets/`
- `docs/assets/asset-manifest.md`
- `docs/assets/asset-ingestion-checklist.md`

## 需要协调后才能触达

- `scripts/rooms/`
- `scripts/main/`
- `tests/`
- `spec-design/`

## 默认不触达

- 核心玩家控制逻辑
- 全局流程治理文档
- git 收口与发布记录

## 输出要求

- 记录新增内容数量、入口、出口和验证路线。
- 资产条目必须说明用途、推荐位置、接入状态和替换优先级。
- 不把未验证玩法包装成正式内容量。
