# 阶段 10：战斗变化与轻量成长循环开发计划

## Summary

以已完成并收口的 `stage9` 为稳定基线，`stage10` 固定采用“`空中攻击` + `第 3 类普通敌人` + `恢复点 + 收集物`”的方案，在不引入正式经济、装备、技能树或大地图系统的前提下，把当前线性小区域推进为“有战斗变化、有轻量成长反馈、有可选支路价值”的下一版可玩切片。

本轮目标不是继续扩基础设施，而是在 `stage9` 已验证成立的房间配置、HUD 快照接口、敌人模板入口和 checkpoint 基线之上，产出一段更完整的内容型阶段成果。本轮继续保留 `分支 + worktree`、阶段开始前的完整 preflight，以及阶段结束前的自动化验证加人工复核收口。

本轮关键选择固定为：

- 新增战斗扩展动作：`空中攻击`
- 战斗变化来源：`第 3 类普通敌人`
- 轻量成长反馈：`恢复点 + 收集物`
- 区域结构：沿用 `stage9` 主区域并新增 `1` 条可选支路与 `1` 个挑战房

## Key Changes

### 战斗变化

- 在当前玩家基线上新增 `空中攻击`，让玩家在跳跃、落地、地面普通攻击之外，获得一个明确可读的空中战斗决策点
- `空中攻击` 只做最小成立版本：
  - 仅 `1` 套空中攻击状态与判定
  - 与当前普通攻击、跳跃、落地、受击状态明确互斥
  - 不额外引入连段、浮空追打、空中冲刺衍生或武器切换
- 新增第 `3` 类普通敌人，要求与 `BasicMeleeEnemy`、`GroundChargerEnemy` 的压力节奏明显不同，但继续复用现有敌人模板与配置资源入口
- 本轮遭遇设计目标不是做 boss 或精英，而是做 `2-3` 组可重复验证的混合遭遇组合，让玩家必须在地面近战、冲锋敌压力和空中出手之间做选择

### 轻量成长循环

- 固定采用 `恢复点 + 收集物` 作为最小成长反馈
- 收集物只承担本区域内的“可选推进价值 / 支路奖励 / 挑战房奖励”职责，不扩成正式货币系统
- 恢复点只做区域内的轻量反馈闭环：
  - 作为玩家阶段推进后的恢复与复位锚点
  - 支持人工试玩时快速判断支路收益是否值得
  - 不扩展为正式长线存档或 meta progression
- 本轮区域中固定加入：
  - `1` 条可选支路
  - `1` 个挑战房
  - 支路与挑战房都必须和收集物 / 恢复收益绑定，而不是只多一间房

### 区域与主流程调整

- `stage10` 不新开第二个大区域，默认在 `stage9` 的内容型阶段成果上继续扩展
- 区域结构固定为：
  - 主线推进段继续成立
  - 新增可选支路
  - 新增挑战房
  - 新增至少 `1` 个更明确体现空中攻击价值的房间或遭遇节点
- `Main` 侧只允许做支撑本轮内容推进所需的最小接口扩展，不重做全局流程框架
- HUD 继续沿用当前稳定快照接口，只补与本轮直接相关的最小信息：
  - 当前收集物提示或数量
  - 恢复点激活提示
  - 空中攻击可读性提示
- 本轮不做 HUD 第三轮重构，只做最小可读性扩展

## Public APIs / Interfaces

- 玩家侧新增最小空中攻击读值 / 状态接口，供测试与 HUD 快照消费
- 敌人侧继续沿用现有基础敌人契约：
  - `receive_attack(hit_direction: Vector2, knockback_force: float) -> void`
  - `defeated`
- 区域侧新增轻量成长相关最小接口：
  - 恢复点激活后的状态读值
  - 收集物状态或数量快照
- HUD 继续统一通过稳定快照接口读取，不允许重新回退到分散字段探测

## Test Plan

- 基线验证继续保留：
  - `godot --headless --path . --import`
  - Stage 1-9 GUT
  - `git diff --check`
- 新增 Stage 10 GUT，至少覆盖：
  - 玩家可在空中进入并退出 `空中攻击`
  - `空中攻击` 与当前地面攻击 / 跳跃 / 受击状态切换不冲突
  - 第 `3` 类普通敌人已接入，且行为节奏与现有两类敌人明显不同
  - 新的混合遭遇可被稳定触发，且不会破坏 `stage9` 的 checkpoint / 房间推进基线
  - 区域内存在 `1` 条可选支路与 `1` 个挑战房，且其奖励与收集物 / 恢复反馈绑定
  - 收集物与恢复点状态可通过稳定接口读取
  - Stage 1-9 既有测试不回归
- 阶段收口前必须补一轮人工复核，不以自动化替代：
  - 真实运行窗口下手动或半手动复核主线推进
  - 至少完整验证一次“主线推进 -> 可选支路 -> 挑战房 -> 回到主线”的区域闭环
  - 确认 `空中攻击` 在至少一个房间 / 遭遇中不是装饰性动作，而有明确玩法价值
  - 确认恢复点与收集物反馈在人工试玩中可看、可玩、可调
  - 记录人工复核结果、失败点、调参重点与是否达到进入 `stage11` 的前置标准

## Assumptions

- `stage10` 默认不再只做“验证点式阶段”，而是产出一个更完整的内容型里程碑
- 玩家扩展动作固定为 `空中攻击`，不在本轮混入第二个新动作
- 敌人变化固定优先做 `第 3 类普通敌人`，不把“普通敌人 + 精英变体”一起塞进本轮
- 轻量成长循环固定为 `恢复点 + 收集物`，不扩展为正式经济系统
- 默认加入 `1` 条可选支路和 `1` 个挑战房；若实现中发现范围失控，也应优先保主线、战斗变化和成长反馈，再裁剪额外内容，而不是削掉 preflight 或人工复核
