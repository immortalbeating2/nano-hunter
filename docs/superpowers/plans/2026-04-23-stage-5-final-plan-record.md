# 阶段 5：教程区垂直切片最终计划记录

> 说明：本文档用于留存本会话中最终确认过的 `stage5` 计划版本。  
> 原始执行用计划仍保留在 [2026-04-22-stage-5-tutorial-vertical-slice.md](/Users/peng8/Desktop/Project/Game/nano-hunter/docs/superpowers/plans/2026-04-22-stage-5-tutorial-vertical-slice.md)。

## Summary

以 `main` 上已完成的 `stage4` 为稳定基线，`stage5` 固定采用“单场景线性教程区 + 低压教学 + 能力教学优先”的方案，把当前已验证的 `move / jump / dash / attack` 串成一条可完整走通的最小主流程。

本轮采用专用 `TutorialRoom`，不继续让 `TestRoom` 承担主流程入口；`TestRoom` 保留为阶段 2/3/4 的回归与调参沙盒。`Main` 从“硬编码加载 TestRoom”调整为“依赖主房间契约”，并默认进入教程区。HUD 本轮采用“提示区 + 战斗面板”的最小形态，其中战斗面板只做展示，不接入完整死亡/恢复循环。

## Key Changes

### 主流程与接口

- 新建专用教程场景 `TutorialRoom`，承载单场景左到右的短流程切片。
- `Main` 不再假定主房间一定是 `TestRoom`，而是依赖统一房间契约：
  - 主房间节点采用通用命名，例如 `Room`
  - 主房间继续暴露 `get_camera_limits() -> Rect2i`
- `Main` 默认实例化 `TutorialRoom`；`TestRoom` 继续独立保留，供回归测试直接加载。
- 教程阻挡目标继续沿用最小受击契约：
  - `receive_attack(hit_direction: Vector2, knockback_force: float) -> void`

### 教程流程与门控

- 教程区固定为 4 段顺序流程：
  - 起始段：移动与跳跃确认
  - 能力段：用地面 `dash` 通过低顶/缺口门槛
  - 战斗段：接近并攻击教程阻挡目标
  - 结束段：出口开启，给出切片完成反馈
- 门控采用低压模型：
  - 不做完整失败重来
  - 允许玩家原地重复尝试
  - 障碍或阻挡目标只做轻量反馈，不接正式扣血/死亡循环
- 本轮不引入正式敌人 AI；战斗教学由教程阻挡目标完成。

### HUD 与可读性

- 新增最小 HUD，固定包含：
  - 教学提示区：当前目标、对应按键提示
  - 战斗面板区：最小生命框位/战斗框位外观、基础状态反馈位
- 战斗面板只做展示：
  - 可显示满状态生命框位
  - 可显示 `dash` 可用性或能力图标高亮
  - 不推进完整生命、死亡、恢复系统
- HUD 的目标是帮助玩家第一次读懂主流程，不把本轮扩成完整前台系统。

### 代理协作方案

- 本轮存在明确的 `subagent / multi-agent` 提效空间，计划中应主动采用，而不是默认主代理单线推进。
- 推荐在实现阶段优先使用 `multi-agent`，按写入范围拆成 3 条并行工作流：
  - 代理 A：主入口与房间契约迁移
    - 负责 `Main`、主房间节点命名、`TutorialRoom` 接入、相机限位契约调整
    - 不负责 HUD 与测试
  - 代理 B：教程区与 HUD
    - 负责 `TutorialRoom` 场景流程、触发区、门控、教程阻挡目标布置、最小 HUD
    - 不改 `Main` 主入口契约
  - 代理 C：测试与文档
    - 负责 Stage 1 现有主房间假设的测试迁移、Stage 5 GUT、新的进度留痕与 `Delegation Log`
    - 不改教程流程实现细节
- 主代理始终负责：
  - 设计确认
  - 任务拆分与边界约束
  - 结果整合
  - 最终验证与合并收口
- 若实现时发现多个子任务需要同时改同一批核心文件，先降级为 `subagent` 或转回主代理串行处理，不为了并行而硬拆。

  - 教程区是否像一个可玩的垂直切片，而不只是 `TestRoom` 换皮

## Assumptions

- `stage5` 的主目标是“教程区短流程串联 + 能力教学可理解”，不是完整游戏循环。
- 本轮默认采用 `分支 + worktree`，并在开始实现前补齐设计文档、实现计划、进度页启动记录。
- `TestRoom` 保留为回归/调参沙盒，不再作为 `Main` 的主流程房间。
- 本轮不引入完整敌人 AI、不接入正式生命/死亡/恢复循环、不展开房间系统重构；若实现中暴露节奏问题，优先回调已有移动、冲刺、攻击参数。
- 只要使用了重要的 `subagent` 或 `multi-agent` 协作，当日日志必须增加 `Delegation Log`。
- 阶段收口时，除验证通过与文档更新外，清理 worktree 前先关闭指向该 worktree 的 Godot/终端/资源管理器，避免 Windows 占用导致目录删除失败。
