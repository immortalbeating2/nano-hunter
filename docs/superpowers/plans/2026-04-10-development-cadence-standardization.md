# 开发节奏标准化实现计划

日期：2026-04-10

## 目标

把“大功能 / 小改动”的默认执行节奏写回项目规范，并同步主线进度文档，作为后续阶段的统一工作方式。

## 实施项

1. 更新 `AGENTS.md`
   - 增加“默认开发节奏”章节
   - 明确大功能和小改动的默认流程
   - 明确 `brainstorming`、`subagent`、`worktree`、提交次数的决策规则
2. 写入设计留痕
   - 在 `spec-design/` 中记录本次标准化的背景、目标、结论和不做项
3. 更新进度文档
   - 在 `docs/progress/timeline.md` 记录本次治理规则标准化
   - 在 `docs/progress/status.md` 记录当前已完成的治理收口
   - 新增当日日志记录本次修改原因、影响和验证结果

## 验证

- 确认 `AGENTS.md` 中新增默认开发节奏章节，且不与现有规则冲突
- 确认 `docs/progress/` 已记录本次治理变更
- 运行 `git diff --check`

## 完成标准

- 后续 session 只靠项目文档即可判断：
  - 当前任务属于大功能还是小改动
  - 是否必须用 `brainstorming`
  - 是否应使用 `仅分支` 还是 `分支 + worktree`
  - 默认应拆成几次提交
