## MCP Debugging

- 初始问题表现：
  - Godot GUI 编辑器已启动，且插件日志显示连接到本地 websocket。
  - 当前 Codex 线程中的 `godot_mcp_pro` 工具却返回 `Godot editor is not connected`。
- 已确认并处理的问题：
  - 阶段 2 worktree 内 `godot_mcp` 版本落后于主工作区，已从 `1.7.2` 对齐到 `1.10.1`。
  - 阶段 2 worktree 的 `project.godot` 移除了若干项目级 autoload，导致 GUI 编辑器启动时一批依赖单例的脚本在编译阶段报错，已恢复。
- 当前结论：
  - 项目侧与插件侧问题已基本收口。
  - 在后续接续中已看到本地 `godot-mcp-pro` node 进程监听 `127.0.0.1:6505`，且外部启动 Godot GUI 后本线程工具仍返回 `Godot editor is not connected`。
  - 在当前干净会话中重新启动指定 Godot 编辑器后，上述现象仍可稳定复现。
  - 当前会话继续向下诊断后，已确认问题不是 Godot 编辑器未连桥，而是线程内 `godot_mcp_pro` transport 与实际活桥错位；一旦清理到只剩活桥，本线程又会落入 `Transport closed` 且不自动重建。
  - 当前已补上一个环境级 workaround：移除桌面端 `godot-mcp-pro` 的固定端口配置，尝试避免新会话继续稳定复现同端口冲突。
  - 当前真正阻塞点仍在桌面端线程与编辑器连接链路，而不是阶段 2 worktree 的项目配置或自动化基线。
  - 阶段 2 的 GUI 手动试玩与参数微调需要在桌面端重建当前线程的 Godot MCP transport 后继续。
