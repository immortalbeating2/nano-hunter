## Godot MCP 流程与脚本策略修正

- 原因：Stage15 MCP 复核中出现“脚本建议重开 / 清理 stale bridge，但 MCP 工具实测可通信”的情况；这说明脚本把“当前工作树编辑器已连接到旧 bridge”误归入了 stale-only 故障。
- 脚本修正：`scripts/dev/godot-mcp-common.ps1` 新增 `ConnectedBridgeAgeUnknown` 推荐动作；只要当前工作树编辑器已经与 bridge 建立连接，就不再仅凭 bridge 启动时间建议清理。
- 入口修正：`scripts/dev/enter-worktree-godot-mcp.ps1` 对 `ConnectedBridgeAgeUnknown` 只提示先用 MCP 只读工具实测；若工具可用继续复核，只有工具缺失、`Transport closed` 或编辑器无法连接时才进入重开 / 清理路径。
- 文档修正：`AGENTS.md` 和 `docs/dev/godot-mcp-pro-connectivity-guide.md` 将 Godot MCP 排障统一为四层判断：会话工具入口、编辑器连接、运行态 autoload、收口清理。
- 流程结论：编辑器场景树可读但运行态截图、输入、脚本执行或 game inspector 不可用时，优先视为 Godot MCP 运行态 autoload 未注入当前游戏实例，不再直接归因到 bridge。
