# Nano Hunter Asset Ingestion Checklist

## 使用方式

每批资产接入前后都要用本清单复核。Stage 12 的目标不是追求正式美术完成，而是让后续资产替换能稳定、可追踪、可回滚。

## 接入前

- [ ] **清单记录**：`asset-manifest.md` 中已经有对应资产 ID、用途、目标路径、来源、授权状态和当前状态。
- [ ] **路径**：文件放在约定目录下，不直接散放到场景或脚本目录。
- [ ] **规格**：尺寸 / 规格与当前原型使用方式匹配，不需要为了资产改核心玩法契约。
- [ ] **授权**：来源和授权状态已记录；临时、AI、免费或购买资产都不能留空。

## Godot 导入

- [ ] **导入**：`godot --headless --path . --import` 能完成，导入日志没有新增阻断错误。
- [ ] **资源引用**：场景引用使用 `res://` 路径；没有绝对路径、用户本机路径或未提交文件。
- [ ] **fallback**：资产不可用时，当前几何占位仍能支撑 demo 运行和测试。

## 运行时显示

- [ ] **显示**：玩家、敌人、HUD、门控、checkpoint、终点图形在默认视口中可见。
- [ ] **碰撞**：视觉外形没有暗示错误的碰撞边界；CollisionShape2D 和 Hurtbox 没有因轻替换改变。
- [ ] **HUD**：图标与文案没有互相遮挡，生命、冲刺、成长 / 目标、完成反馈仍可读。
- [ ] **流程理解**：checkpoint、门控、终点和重开入口更容易理解，而不是引入新的误导。

## 验证与留痕

- [ ] **自动化**：Stage 12 专项 GUT 通过。
- [ ] **回归**：Stage 1-12 全量 GUT 通过。
- [ ] **人工复核**：完整 demo、支路、挑战房、失败 / 重来、完成反馈与重开入口已检查。
- [ ] **文档**：当日日志记录做了什么、为什么做、影响范围、验证结果、风险与下一步建议。

## 当前 Stage 12 第一批结果

- 玩家、3 类敌人、HUD、终点房、攻击 slash 与命中 spark 只接入轻量视觉标记，不改变碰撞、AI、房间切换或 checkpoint 契约。
- 攻击 slash 与命中 spark 已有 Stage 12 回归测试保护，后续若改成正式动画或粒子，需要继续保证不参与判定。
